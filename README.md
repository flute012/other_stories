# 課程學習活動網站UI設計展示
![專案示意圖](https://via.placeholder.com/800x400/4A90E2/FFFFFF?text=Interactive+Learning+System)

## 📖 專案緣起

在原有的設計中，使用者需要經過**三層頁面**才能進入功能：
```
冊次選擇頁 → 課次選擇頁 → 功能選擇頁 → 實際功能
```
這種設計導致**新使用者流失率高達 70%**。

### 解決方案

將三層頁面整合為**單一介面**，使用者可以在同一個畫面中：
- ✅ 瀏覽並切換冊次
- ✅ 快速導覽課次
- ✅ 直接選擇功能

**目標：** 使用者只在目錄頁即可找到目標功能，大幅降低操作路徑，減少用戶網頁載入負擔。

---

## ✨ 功能特色

### 1. 三維導覽系統
- **冊次切換**：左側垂直列表，支援 4 冊課程
- **課次導覽**：頂部顯示當前與相鄰課次，支援左右滑動與快速跳轉
- **功能選擇**：8 個功能按鈕，清晰的視覺分類

### 2. 多種互動方式
- **觸控滑動**：左右滑動切換課次（支援手機）
- **快速導航點**：4 個圓點，每點代表 3 課，點擊快速跳轉
- **課次標籤**：直接點擊課次圓圈切換


## 🎨 設計理念

### 1. 減少認知負擔
- **一致的視覺語言**：圓形元素貫穿整個介面（課次、導航點、功能圖示）
- **色彩編碼**：8 個功能用不同的漸層色區分
- **視覺層次**：當前課次放大 + 陰影，明確指示狀態

### 2. 提升操作效率
- **快捷方式**：導航點可跳過 3 課，減少滑動次數
- **即時回饋**：所有互動都有視覺回饋（過渡動畫、hover 效果）
- **防止誤觸**：滑動閾值設定，避免輕觸誤判

### 3. 資料結構
**使用階層式結構**：

booksData = [
  {
    id: 1,
    name: '第一冊',
    lessons: [...]  // 12 課
  }
]


**優點**：
- 易於擴充（增加冊次或課次）
- 切換冊次時可完整載入該冊資料
